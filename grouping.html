Topic 6

Grouping Tags

The <div> and <span> tags are used to group HTML elements together so they can be styled or controlled as a unit using CSS and JavaScript.

Why Group Elements?

By themselves, <div> and <span> do not change how content looks. Their power comes from pairing them with CSS classes or IDs to apply styles to groups of elements at once — enabling layouts, coloured sections, and more.

1. The <div> Tag — Block-Level Container

The <div> (Division) tag is a block-level element. It creates a rectangular block that stretches the full width of its container and starts on a new line.

It is used to group large sections of content such as headers, footers, sidebars, and main content areas.

HTML — div Example
<div style="background-color: lightblue; padding: 20px;">
  <h2>This is inside a div</h2>
  <p>The div groups the heading and paragraph together.</p>
</div>

<div style="background-color: lightyellow; padding: 20px;">
  <p>This is a second, separate div block.</p>
</div>
Output

This is inside a div

The div groups the heading and paragraph together.

This is a second, separate div block.

2. The <span> Tag — Inline Container

The <span> tag is an inline element. It does not create a new line — it stays within the flow of text. It is used to style a specific word or phrase inside a paragraph.

HTML — span Example
<p>
  In web development, <span style="color: red; font-weight: bold;">HTML</span>
  handles structure, <span style="color: blue; font-weight: bold;">CSS</span>
  handles design, and <span style="color: green; font-weight: bold;">JavaScript</span>
  handles behaviour.
</p>
Output

In web development, HTML handles structure, CSS handles design, and JavaScript handles behaviour.

div vs span — Key Differences

Feature<div><span>
Display TypeBlock-level elementInline element
Starts on a new line?Yes — always starts on a new lineNo — stays in the text flow

Using div and span with CSS Classes

The real power of div and span comes when combined with CSS classes or IDs:

HTML + CSS — Using class attribute
<!-- HTML -->
<div class="card">
  <h3>Topic Title</h3>
  <p>This card uses a <span class="highlight">CSS class</span> for styling.</p>
</div>

/* CSS */
.card {
  background: #f5f5f5;
  border-radius: 8px;
  padding: 20px;
  border-left: 4px solid #006644;
}
.highlight {
  background-color: yellow;
  font-weight: bold;
}
Output

Topic Title

This card uses a CSS class for styling.

Semantic HTML5 Grouping Elements

HTML5 introduced more descriptive alternatives to <div> that carry meaning:

TagPurpose
<header>Top section of a page or section
<nav>Navigation menu container
<main>Main content area of the page
<section>A thematic group of content
<article>Self-contained content (blog post, news)
<aside>Side content, like a sidebar
<footer>Bottom section of a page or section
💡 Best Practice

Use semantic HTML5 tags (<header>, <nav>, <main>, etc.) instead of generic <div> tags when possible. This improves accessibility and helps search engines understand your page structure.
