Topic 3

Form Tags

HTML forms are used to collect user input. They contain various interactive controls like text fields, checkboxes, radio buttons, and submit buttons.

Core Form Tags

Tag Purpose
<form>Container for all form elements
<input>Creates various types of input fields (most versatile tag)
<label>Associates a label with a form element for accessibility
<textarea>Multi-line text input area
<select>Creates a dropdown list
<option>Defines an item inside a <select> dropdown
<button>A clickable button
<fieldset>Groups related form elements together
<legend>Provides a title for a <fieldset>
<datalist>Provides predefined suggestions for an input field

Input Type Values

The <input> element uses a type attribute to determine what kind of control is rendered:

type=DescriptionExample
textSingle-line text field<input type="text">
passwordPassword field (characters hidden)<input type="password">
emailEmail address (validates format)<input type="email">
numberNumeric input with optional min/max<input type="number" min="1" max="10">
dateDate picker<input type="date">
colorColor picker<input type="color">
checkboxTick box — can select multiple<input type="checkbox">
radioRound button — select one from a group<input type="radio" name="group">
fileFile upload button<input type="file">
rangeSlider control<input type="range" min="0" max="100">
submitSubmits the form<input type="submit" value="Submit">
resetResets all form fields<input type="reset" value="Reset">
urlURL input (validates format)<input type="url">
telTelephone number<input type="tel">
timeTime picker<input type="time">
monthMonth and year selector<input type="month">

Complete Form Example

HTML — Registration Form
<form action="submit.html" method="post">

  <fieldset>
    <legend>Personal Information</legend>

    <label for="name">Full Name:</label>
    <input type="text" id="name" name="name" placeholder="Enter your name">

    <label for="email">Email:</label>
    <input type="email" id="email" name="email">

    <label for="dob">Date of Birth:</label>
    <input type="date" id="dob" name="dob">
  </fieldset>

  <label for="gender">Gender:</label>
  <input type="radio" name="gender" value="male"> Male
  <input type="radio" name="gender" value="female"> Female

  <label for="country">Country:</label>
  <select id="country" name="country">
    <option>Rwanda</option>
    <option>Kenya</option>
    <option>Uganda</option>
  </select>

  <label for="msg">Message:</label>
  <textarea id="msg" rows="4" cols="40"></textarea>

  <input type="checkbox" name="terms"> I agree to the terms
  <br>
  <input type="submit" value="Submit">
  <input type="reset" value="Reset">

</form>
Live Working Output
Personal Information

Male   Female

I agree to the terms

💡 Tip

Always use <label> with a matching for attribute that equals the input's id. This improves accessibility and allows users to click the label to focus the input.
